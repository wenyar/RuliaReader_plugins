# 漫画屋 Rulia 插件

在 Rulia 里阅读 e728.com 的漫画。

数据来源：https://www.e728.com/

## 接口与逻辑说明

- 列表页：默认读取 `GET https://www.e728.com/category/`，分页为 `/category/page/{page}`；分类筛选读取 `/category/tags/{id}`，状态筛选读取 `/category/finish/{id}`，排序读取 `/category/order/{key}`，多条件会组合成同一个分类路径。插件只实现源站有真实分页的分类浏览，不再提供排行榜、最新、推荐、完结等无分页入口。源站每页约 20 本，插件按 Rulia 传入的 `pageSize` 换算并合并源站分页，返回结构只保留 `{ list }`。
- 筛选：提供写死在插件里的完整类型列表、连载状态和排序方式，均为分类浏览内部筛选；打开筛选时不请求源站。
- 搜索页：使用 `GET /search?key={keyword}`，从 `.comic-item` 结果块解析标题、详情 URL、封面和作者。源站会忽略搜索分页参数，所以插件在搜索续页或重复请求第一页时直接返回空列表。
- 详情页：详情 URL 形如 `/comic_10627.html`，解析 `.comic-meta-info h1`、`og:image`、`.comic-description p`、标签和 `.chapter-list a[href*="/chapter_"]`。
- 章节图片：章节 URL 形如 `/chapter_10627_970667.html`，章节页在 `.comic-content` 中直接内嵌 `img.comic-image`，插件按页面顺序返回图片 URL。
- 图片地址：封面来自 `comic.5um.net`，章节图来自 `manhua.5um.net`；预检确认样例图片不带特殊 `Referer` 也能直连，`getImageUrl` 直接返回原地址。已知坏封面会返回文本或非图片响应；列表页和详情页都使用源站 logo 图片占位，不使用其他漫画的默认封面替代。
- 缓存与兜底：详情数据和章节图片列表做内存缓存，减少重复请求。源站请求有限重试；分类列表合并多页时，若后续源站页偶发失败，会返回已解析出的部分结果，避免一直等待。

## Payload

插件不自定义章节 payload，章节 URL 直接使用源站 `/chapter_{comicId}_{chapterId}.html` 地址。

## 验证样例

- 分类列表：`https://www.e728.com/category/` 返回 `凤逆天下`、`绝世武神` 等条目。
- 搜索关键词：`斗破`，返回 `斗破苍穹` 相关结果。
- 详情页样例：https://www.e728.com/comic_10627.html
- 章节样例：https://www.e728.com/chapter_10627_970667.html
- 章节图片样例：`https://manhua.5um.net/colatj/meiziyukexue/2/7970f56d408c76531e0b6f8ef0c5fc0b.webp`，直连返回 `image/webp`。

## 已知限制

- 源站页面包含广告脚本和 Cloudflare 统计脚本，插件只解析漫画相关 HTML，不请求广告接口。
- 搜索页当前样例只显示一页结果；源站会忽略搜索分页参数，插件按单页搜索处理，避免 Rulia 无限加载重复结果。
- 如果 `comic.5um.net` 或 `manhua.5um.net` 图片 CDN 变更，需同步更新图片解析规则；若后续出现新的坏封面，也需要补充兜底表。
- 付费或权限章节会按源站页面返回结果处理；若页面不内嵌图片，插件会报“无法解析章节图片”。
